## Needs investigation (gate severities — read before acting)

- [x] **`X-05` registry reload atomicity — RESOLVED 2026-06-01 (mostly moot).** Read `cmd/mockagents/start.go` + `internal/server/watcher.go`. Findings:
  - **Boot** (`start.go:88-101`): a fresh `AgentRegistry` is built and `Register`-ed sequentially **before** the server serves, so no concurrent readers exist during initial load.
  - **Hot-reload** (`watcher.go reloadFile:169`): mutates the live registry in place — but via a **single `Register` per changed file**, and `AgentRegistry` is a `sync.RWMutex` (`Register` holds `Lock`, every read holds `RLock`). So each reload is atomic w.r.t. readers → **the "torn reads / half-updated state" concern is NOT real.**

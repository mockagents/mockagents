# MockAgents v0.4.0 — first public release 🎉

**Mock the OpenAI, Anthropic & Gemini APIs — and MCP & A2A agents — with zero LLM calls. Fast, free, deterministic, offline.**

MockAgents is a single pure-Go binary that's a **drop-in replacement** for the OpenAI, Anthropic, and Google Gemini HTTP APIs, plus MCP and A2A agents. Point your app's base URL at it and your existing SDK code just works — returning **deterministic** canned responses, simulated tool calls, and real SSE streams, with **no real LLM calls**. Your agent tests stop costing a dollar a run, stop flaking on model updates, and run fully offline.

This is the **first public release**. Everything below ships in `v0.4.0`.

```bash
docker run -p 8080:8080 mockagents/mockagents      # run the mock (no prereqs)
export OPENAI_BASE_URL=http://localhost:8080/v1    # the ONLY change to your app
export OPENAI_API_KEY=mock
python my_existing_app.py                          # it works — free, offline, deterministic
```

## ✨ Headline capabilities

### Wire-compatible provider surfaces
- **OpenAI** — Chat Completions, the **Responses API** (the default Agents-SDK transport: typed output items, the full `response.*` SSE event ladder, stateful `previous_response_id`), **Conversations API** (the post-Assistants stateful stack), Embeddings, Structured Outputs (`json_schema` strict mode + refusals), and Moderations.
- **OpenAI Realtime API over WebSocket** — test **voice agents** offline: connect over a WebSocket, send the Realtime events, and get back the full streamed response ladder with a deterministic transcript (from your scenarios) and synthesized audio — no TTS, no audio model, fully reproducible.
- **Files + Batch API** (OpenAI) and **Anthropic Message Batches** — run the full upload/submit → poll → retrieve flow offline.
- **Anthropic** — Messages, `count_tokens`, prompt-caching usage, extended-thinking blocks.
- **Google Gemini** and **Azure OpenAI** URL surfaces.
- **Vision input** parsing (OpenAI `image_url`/data-URL, Anthropic base64/url) with a `has_image` scenario rule.

### Agent-protocol mocks
- **MCP (Model Context Protocol)** — mock MCP servers over Streamable HTTP + stdio; **conformance-validated** against the official suite. `mockagents mcp --manage` even exposes the agent write API *as* MCP tools, so an MCP client can manage your fixtures conversationally.
- **A2A (Agent2Agent)** — mock Google's agent-to-agent protocol with `kind: A2AServer` + `mockagents a2a`: serves the Agent Card and answers `message/send` / `tasks/get` / `tasks/cancel`.

### Built for testing
- **Deterministic by design** — same request, same response, every run.
- **Scenario matching** (content / regex / turn) — assert *which* path fired, not just the text.
- **Agent-trajectory assertions** in `mockagents test` — `tool_call` (name + partial args), `tool_call_count`, `tool_call_sequence`, and `node_sequence` (pipeline trajectory): deterministic checks for the wrong-tool / wrong-count / wrong-order bugs that belong on every PR.
- **Chaos & fault injection** — latency, errors, rate limits, connection-level faults; streaming-timing physics (TTFT/ITL); **hallucination fixtures** that return confidently-wrong output on demand.
- **Multi-agent pipelines** (`kind: Pipeline`) — sequential, parallel, and graph topologies.
- **Record & replay** and **contract testing** (extract + diff breaking changes in CI).

### Batteries included
- **Three SDKs** (Python / TypeScript / Go) with streaming helpers; the Go SDK runs an engine **in-process** with no subprocess.
- **Test-runner helpers** — a `pytest` plugin and `@mockagents/vitest` (+ Jest) that auto-spawn the server and redirect the provider SDKs.
- **Web console**, **Docker image**, **Helm chart**, and **GitHub Actions / GitLab CI** templates.
- **Enterprise surface** (opt-in) — multi-tenancy, RBAC, audit log, per-tenant quotas, SSO/OIDC, OpenTelemetry tracing, Postgres or SQLite.

## 🚀 Quick start

```bash
mockagents init my-project && cd my-project   # scaffold an example agent
# ...or start from a curated pack:
mockagents init my-bot --template customer-support   # see --list-templates
mockagents start                              # prints your base URL + a ready-to-paste snippet
```

Then swap your app's base URL (`OPENAI_BASE_URL` / `ANTHROPIC_BASE_URL` / `GOOGLE_GEMINI_BASE_URL`) to the printed address and run your tests — no other changes.

## What's notable in 0.4.0

The release rounds out four net-new tracks aimed at where agent development is heading in 2026:

- **Voice-agent testing** — the OpenAI Realtime API over WebSocket (the first WebSocket transport in the mock).
- **The post-Assistants stack** — the OpenAI Conversations API (the Assistants API sunsets 2026-08-26).
- **Deeper agent-behavior testing** — tool-call/trajectory assertions in the test runner.
- **Agent-to-agent** — A2A mock servers alongside the existing MCP mocks.

…plus Anthropic Message Batches, an MCP conformance badge, and the SDK/CI distribution wiring.
